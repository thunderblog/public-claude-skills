# /token-saver — トークン消費削減スキル

## 概要

ClaudeCodeのコスト爆発の根本原因は「コンテキストの肥大化」。
このスキルは3つのアクションでそれを構造的に防ぐ。

---

## トリガー

- `/token-saver`
- 「コストを節約したい」「トークンが多い」「セッションをリセットしたい」
- 「CLAUDE.mdを見直したい」「コストが気になる」

---

## ステップ 1：モードを判断する

引数またはユーザーの発言からモードを判断する。

| モード | トリガーワード | 内容 |
|--------|--------------|------|
| **handoff** | 「リセット」「引き継ぎ」「まとめて」`/token-saver handoff` | 現在のセッション状態を次回引き継ぎ用にまとめる |
| **audit** | 「監査」「見直し」「CLAUDE.md」`/token-saver audit` | CLAUDE.mdの肥大化を診断して削減提案する |
| **check** | 引数なし・「節約」「チェック」 | 6つの対策チェックリストを表示 |

判断できない場合は **check** モードで動作する。

---

## モード A：handoff（セッション引き継ぎ）

現在の会話コンテキストを要約し、次のセッションにそのまま貼れる引き継ぎドキュメントを生成する。

### 出力フォーマット

```
## セッション引き継ぎ（{日付}）

### 現在取り組んでいるタスク
- （今やっていること）

### 完了済み
- （完了したこと）

### 次にやること
- （次のステップ）

### 重要な決定事項
- （このセッションで決まったこと）

### 関連ファイル・パス
- （作業中のファイルパスなど）

### 引き継ぎメモ
（注意点・文脈・前提など）
```

生成後、以下を案内する：

```
このサマリーを新しいセッションの最初に貼り付けて再開してください。
新鮮なコンテキストで品質とコスト両方が改善されます。
```

---

## モード B：audit（CLAUDE.md監査）

グローバルおよびプロジェクトの `CLAUDE.md` を読み込み、肥大化リスクを診断する。

### 手順

1. グローバルの `CLAUDE.md` を読み込む（Linux/Mac: `~/.claude/CLAUDE.md`、Windows: `%USERPROFILE%\.claude\CLAUDE.md`）
2. カレントディレクトリの `CLAUDE.md`（プロジェクト）があれば読み込む
3. 以下の観点で診断する

### 診断観点

| カテゴリ | 削除推奨の内容 |
|----------|--------------|
| AIが既知の情報 | 一般的なベストプラクティス・コーディング規約 |
| 歴史・経緯 | 「以前は〜だったが」「〜という経緯で」 |
| 過剰な説明 | 1行で書けるルールに3行以上かけている箇所 |
| 重複 | 同じルールが複数箇所に書かれている |

### 出力フォーマット

```
## CLAUDE.md 監査結果

### グローバル CLAUDE.md
- 行数: XX行（推定 XXXトークン）
- 削減推奨箇所:
  - [行XX-XX] 「〜」→ AIが既知のため削除可
  - [行XX-XX] 「〜」→ 1行に圧縮可能

### 推定削減効果
現在: 約XXXトークン → 削減後: 約XXXトークン（XX%削減）

### アクション
削減を実行しますか？ → 「はい」で対象箇所を修正します
```

---

## モード C：check（コスト削減チェックリスト）

6つの対策を現状確認つきで表示する。

```
## ClaudeCode コスト削減チェックリスト

① セッションリセット（30-45分ごと）
  → 長いセッションはコンテキスト肥大化の主因。/token-saver handoff で引き継ぎサマリーを生成できます。

② CLAUDE.mdを短く保つ
  → ビルドコマンド・制約・禁止事項だけ残す。/token-saver audit で診断できます。

③ SkillsのProgressiveDisclosure
  → 情報はCLAUDE.mdでなくSkillに分離。必要なときだけロードされます。
  　（25,000トークン → 955トークンになった事例あり）

④ Prompt Cachingを壊さない
  ✗ System Promptにタイムスタンプを入れる
  ✗ セッション途中でツールを追加・削除する
  ✗ セッション途中でモデルを切り替える

⑤ タスクの複雑度でモデルを使い分ける
  → コード・デバッグ: 中堅モデル
  → メール・スケジュール: 軽量モデル
  → 戦略・ブレスト: 最上位モデル

⑥ コンテキスト汚染を構造的に防ぐ
  コンテキスト管理 ＝ コスト管理 ＝ 品質管理

どれかを今すぐ実行しますか？ → 番号を教えてください
```

## 使用例

### 例A：セッション引き継ぎ

**ユーザー:** `/token-saver handoff`

**Claude:** 現在の会話を解析して引き継ぎサマリーを生成し、次セッションへの貼り付け手順を案内する。

---

### 例B：CLAUDE.md監査

**ユーザー:** `CLAUDE.mdが長くなってきた /token-saver audit`

**Claude:** CLAUDE.mdを読み込み、削除推奨箇所と削減トークン数を提示。承認後に修正する。

---

### 例C：チェックリスト確認

**ユーザー:** `/token-saver`

**Claude:** 6つの対策チェックリストを表示し、次のアクションを案内する。
